🛠️ Компоновка системы

Механика

  • 4 мини-мотора с энкодерами
  • Алюминиевая рама
  • Колеса 80мм

Электроника

  • Raspberry Pi 4 (основной компьютер)
  • ESP32-S3 (контроллер двигателей)
  • Драйвер HobbyWing QuicRun 1060 60A
  • LiPo аккумулятор 12V 5000mAh

Сенсоры

  • Лидар RPLIDAR A2M8 (8м, 360°)
  • Камера ZED
  • IMU MPU6050

⚡ Электрическая схема

Схема подключения компонентов

RPi4 → ESP32-S3 → Драйвер → Моторы

Важно

Используйте понижающий преобразователь 12V → 5V для питания RPi4

Защита

Обязательно установите предохранитель на 10А в цепь питания

🚀 Порядок запуска

Режим ручного управления

1 ssh ros2@Ip_adress — подключение к Raspberry Pi
2 export ROS_DOMAIN_ID=42 — установка domain ID
3 ros2 run teleop_twist_keyboard teleop_twist_keyboard — управление роботом
1 Ввести пароль — авторизация
2 export ROS_DOMAIN_ID=42 — установка domain ID
3 ros2 launch robot_start start.launch.py — запуск узлов робота

⚠️ Возможные поломки и неисправности

Не включается

Причина: Разряжен аккумулятор

Решение: Зарядить LiPo (12.6V полный заряд)

Нет связи с ESP32

Причина: Проблема с UART

Решение: Перепрошить ESP32, проверить пины

Лидар не работает

Причина: USB подключение/драйвер

Решение: Проверить lsusb, перезапустить sllidar_ros2

Нет связи ROS2

Причина: ROS_DOMAIN_ID разный

Решение: Установить одинаковый ID на всех устройствах

Важное примечание

В качестве базовой платформы изначально использовался NVIDIA Jetson TX2. Однако в процессе разработки возникли сложности с установкой пакета NAV2 из-за ограниченной поддержки ARM64-архитектуры (aarch64) для некоторых зависимостей. Пакет ros-jazzy-navigation2 не имеет готовых сборок для Jetson TX2 в ROS2 Jazzy.
